## 🗦 Technical Architecture

This system is built from the ground up prioritizing scalability, fluid character momentum physics, and easy integration.

* 🧩 **100% Blueprint-Based:** Designed cleanly without C++ code dependencies, ensuring accessibility and rapid iteration for developers.
* 📦 **Modular & Component-Based:** Built using isolated Actor Components. The entire weapon generation and traversal logic can be attached to any character or controller seamlessly without hardcoding dependencies.
* 📊 **Data-Driven Configuration:** Everything from weapon range, visual design, usage limits, to zipline behavior is configured from centralized structures, removing technical complexity.

---

## ✨ Main Features

* **Customizable Zipline Creation Weapon:** Mechanics to shoot anchors and dynamically build lines.
* **Physics & Momentum Control:** Built-in systems for acceleration, inertia, and direction changes.
* **Obstacle Detection System:** Automated safety checks to avoid clipping during traversal.
* **Dynamic Weapon HUD:** Real-time feedback for ammunition, active lines, and range tracking.
* **VFX, SFX & Animation Registry:** Complete setup for zipline meshes, equip animations, and audio-visual triggers.
